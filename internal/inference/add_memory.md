Save a memory for future reference. Memories persist across sessions and appear in your system prompt every turn — so be selective.

When to save:
- The user corrects you or validates an unexpected approach
- You discover the user's role, preferences, or domain expertise
- A project decision, constraint, or deadline that lives outside the codebase
- Locations of external resources (dashboards, ticket boards, chat channels)

When NOT to save:
- Things you can figure out by reading the code or running git log
- Information that's already in your agent instructions or project docs
- Short-lived details that only matter for the current session

Content should be a single concise line (1-3 sentences). No newlines. Older memories are evicted when the limit is reached.
